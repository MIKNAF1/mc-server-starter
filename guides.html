Guides

Simple steps for beginners. If you want, we can later add screenshots.

Paper setup (recommended)

  1. Download Paper from the official site: papermc.io/downloads
  2. Put the downloaded jar into your server folder and rename it to server.jar.
  3. Run start.bat once (it will generate files and exit).
  4. Open eula.txt and change eula=false to eula=true.
  5. Run start.bat again. Your server should start.

Tip: If your server lags, reduce view-distance, lower player count, or allocate more RAM (but not too much).

Vanilla setup

  1. Download the official server jar: minecraft.net server download
  2. Rename it to server.jar and place it into your folder.
  3. Run start.bat, accept the EULA, then run again.

Fabric setup

  1. Follow Fabric’s server instructions: fabricmc.net/use/server
  2. After you have the server jar/launch file ready, ensure your final runnable jar is named server.jar in your folder.
  3. Run start.bat, accept EULA, run again.

Forge setup

  1. Download Forge from: files.minecraftforge.net
  2. Forge sometimes uses a generated run script. For simplicity, create/point to a runnable server.jar in your folder.
  3. Run start.bat, accept EULA, run again.

Forge setups can vary by version. If you want, tell me your exact Forge version and I’ll tailor this page.

Port forwarding basics

  • Local play (same Wi‑Fi) usually works without port forwarding.
  • For friends outside your network, forward TCP port 25565 to your PC’s local IP.
  • Allow Java/Minecraft in Windows Firewall when prompted.
  • Share your public IP address (or use a DDNS service).

If you tell me your router brand (Dialog, SLT, etc.), I can guide you with the exact menu steps.

Common fixes

  • “Java not found”: install Java (Adoptium Temurin is a popular choice) and try again.
  • Server closes instantly: check logs/latest.log and confirm EULA is true.
  • Can’t connect from outside: port forward + firewall + correct public IP.
  • Lag: reduce view-distance, remove heavy plugins/mods, or lower max players.
